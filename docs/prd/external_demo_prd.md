# 外部真实接入替代验证 PRD

## 背景

为了在没有公司内网元宝环境权限的情况下验证测试 Agent 平台的真实接入能力，建设一个本地真实 Web Demo 业务系统，并通过 GitHub Actions、GitHub Issues、Markdown PRD 完成外部可验证闭环。

## 功能点

1. 用户可以打开登录页并进入“我的”页面。
2. 用户可以进入搜索页，输入 `Yuanbao` 并看到商品或服务结果。
3. 用户可以将搜索结果加入购物车，并进入下单确认。
4. 用户可以进入设置页关闭通知开关，重新进入后状态应保持关闭。

## 边界条件

- 搜索关键词为空时展示默认结果。
- 搜索关键词不存在时结果列表为空但页面不报错。
- 通知开关关闭后重复点击关闭不应恢复开启。
- 下单接口返回订单号后，报告中应保留订单状态。

## 验收标准

- GitHub Actions 可作为开发自测触发来源。
- GitHub Issues 可作为 BUG 回归来源和结果回写目标。
- Markdown PRD 可作为需求测试输入来源。
- 本地 Web Demo 必须通过真实 HTTP 请求访问，不允许只用内存数据假装业务系统。
